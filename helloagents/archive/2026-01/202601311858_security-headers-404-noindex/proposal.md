## 1. 需求

### 背景
当前项目缺少统一的安全响应头和分享访问失败的统一 404 策略，存在信息泄露与安全基线不足的问题。

### 目标
- 全站响应添加 CSP、X-Content-Type-Options、Referrer-Policy 等安全头
- 分享访问失败/权限失败统一返回 404
- 分享页包含 noindex/noarchive 元信息
- 覆盖安全头与分享 404 的测试

### 约束条件
```yaml
时间约束: 无
性能约束: 不能显著影响响应性能
兼容性约束: Next.js 16 App Router
业务约束: 不引入新业务权限逻辑
```

### 验收标准
- [ ] 核心页面响应包含安全头
- [ ] 分享访问失败场景统一返回 404
- [ ] 分享页包含 noindex meta
- [ ] 单元测试覆盖安全头输出
- [ ] 集成测试覆盖分享访问失败 404
- [ ] 边界测试覆盖不同路由/方法的头部一致性

---

## 2. 方案

### 技术方案
- 使用 `next.config.ts` 的 `headers()` 设置全站安全头
- 添加 `middleware.ts` 为分享路由补充 noindex/noarchive 头部控制
- 在分享路由中校验 token 格式，失败直接 404
- 通过 `metadata.robots` 为分享页输出 noindex/noarchive meta

### 影响范围
```yaml
涉及模块:
  - share: 分享页与访问失败策略
  - app: 路由/中间件与全局配置
预计变更文件: 6-8
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| CSP 过严导致资源被拦截 | 中 | 采用可用且保守的 CSP，必要时按需放开 |
| 分享功能未完备导致误判 | 低 | 仅基于 token 格式做失败 404，后续接入真实校验 |
