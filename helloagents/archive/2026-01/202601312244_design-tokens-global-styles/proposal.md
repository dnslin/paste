## 1. 需求

### 背景
落地 UI 设计指南中的配色、圆角、字体与间距规范，建立可复用的全局设计令牌，保证后续页面与组件风格一致。

### 目标
- 建立全局设计令牌（颜色/圆角/阴影/间距/动效）并可复用
- 配置字体与基础排版基线（像素字体仅作点缀）
- 定义全局布局与间距系统（单栏居中 + 最大宽度）
- 输出设计令牌文档，便于后续组件复用
- 覆盖对应测试（CSS 变量、主题加载、字体回退）

### 约束条件
```yaml
时间约束: 无
性能约束: 字体文件轻量，避免大体积字体
兼容性约束: 现代浏览器（Chrome/Firefox/Safari/Edge 最新两个版本）
业务约束: V1 只做浅色；像素字体仅用于点缀场景
```

### 验收标准
- [ ] 设计令牌覆盖 UI 设计指南核心规范
- [ ] 页面基础排版与间距一致
- [ ] 像素字体仅用于点缀场景
- [ ] Unit tests: CSS 变量命名与值验证
- [ ] Integration tests: 基础页面能正确加载主题
- [ ] Edge cases: 不同浏览器字体回退

---

## 2. 方案

### 技术方案
- 在 src/app/globals.css 定义设计令牌（颜色/圆角/阴影/间距/动效/布局）
- 在 src/app/layout.tsx 配置 DM Sans + JetBrains Mono + Silkscreen 字体变量
- 提供像素字体专用样式类，避免全局误用
- 基础布局通过全局容器类统一最大宽度与边距
- 新增 docs/design-tokens.md 输出可复用令牌说明
- 使用 node:test 添加主题相关测试

### 影响范围
```yaml
涉及模块:
  - app: 全局样式与布局、字体
  - docs: 设计令牌文档
  - tests: 主题与字体相关测试
预计变更文件: 6-8 个
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 全局样式影响现有页面布局 | 中 | 增量改动 + 测试覆盖 + 保持语义类名最小化 |
| 字体加载不符合轻量要求 | 低 | 仅加载 Latin 子集并限制像素字体使用范围 |
| Tailwind 与自定义 CSS 冲突 | 低 | 保持 Tailwind 导入不变，避免重置其核心层 |
