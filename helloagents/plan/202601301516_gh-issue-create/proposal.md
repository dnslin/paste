## 1. 需求

### 背景
需要将 PRD 拆解为 EPIC 与子 Issue，并在 GitHub 新仓库中创建里程碑与 Project，便于后续开发管理。

### 目标
- 初始化本地 Git 仓库并完成首个提交
- 在 GitHub 创建公开仓库并推送
- 基于 PRD + UI 指南生成 EPIC/子 Issue，创建里程碑与新 Project

### 约束条件
```yaml
时间约束: 无
性能约束: 无
兼容性约束: 无
业务约束: 必须按 EPIC/子 Issue 关系拆分
```

### 验收标准
- [ ] GitHub 仓库创建成功且包含首个提交
- [ ] 里程碑与 Project 创建成功（不复用旧 Project）
- [ ] EPIC 与子 Issue 已创建并互相关联

---

## 2. 方案

### 技术方案
使用 git 初始化仓库并提交；使用 gh CLI 创建 GitHub 仓库；使用 gh-issue-create 脚本批量创建里程碑、标签、Project 和 Issues。

### 影响范围
```yaml
涉及模块:
  - repo: Git 仓库初始化与远端创建
  - pm: 里程碑与 Project 创建
  - issue: EPIC/子 Issue 创建
预计变更文件: 4+
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 仓库已存在导致创建失败 | 中 | 创建前检查或使用既有仓库 |
| 标签缺失导致脚本失败 | 低 | 在配置中显式创建所有用到的标签 |
| 依赖顺序导致 Issue 创建阻塞 | 低 | 为依赖任务设置 dependencies |

**日期**: 2026-01-30
